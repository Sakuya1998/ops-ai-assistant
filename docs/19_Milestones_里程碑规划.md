## M1: 编排骨架与对话诊断（2周）

### 目标
建立Eino编排框架，实现基础对话诊断能力

### 交付物
- [ ] 项目结构搭建完成
- [ ] Eino Graph基础框架
- [ ] `/api/v1/chat` 接口可用
- [ ] 基础工具：query_metrics, query_logs
- [ ] 单元测试覆盖率 > 60%

### 验收标准
用户输入"为什么payment-service延迟升高"，系统返回结构化诊断结果

## M2: 告警接入与自动分诊（2周）

### 目标
实现告警Webhook接入，自动生成Incident和Top3根因假设

### 交付物
- [ ] `/api/v1/alerts/webhook` 接口
- [ ] Alert解析与去重聚合
- [ ] RCA引擎基础实现
- [ ] Incident/Hypothesis/Evidence数据模型
- [ ] 集成测试通过

### 验收标准
Webhook接收告警后30秒内生成Incident，包含Top3假设和证据链

---

## M3: 多智能体并行诊断（3周）

### 目标
实现Supervisor-Specialist多智能体协作

### 交付物
- [ ] SupervisorAgent实现
- [ ] NetworkAgent实现

- [ ] AppAgent实现

- [ ] 并行调度与结果聚合

## M4: 修复执行与审批回滚（2周）

### 目标
实现可控自动化修复能力

### 交付物
- [ ] PolicyGuard策略门禁
- [ ] 高危动作审批流程
- [ ] 执行前快照与后验证
- [ ] 回滚机制

### 交付物
- [ ] VikingDB集成

- [ ] OpenTelemetry全链路追踪
